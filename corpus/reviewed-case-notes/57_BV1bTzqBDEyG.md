## User Facts Extracted

- 连线人不是老板，是店内员工，32 岁，自费买课来问诊，目标是帮老板改进生意、保住门店。后段自述负债 100 多万，原因是买房，不是该店经营负债。
- 门店在重庆；后段补充属于璧山区，离沙坪坝大学城较近。位置是山顶小院/山上院子，附近有爬山路线和一个小景区名 ASR 不稳；下山约四五公里到大学城，水泥路通达，车能开上来。
- 业态是下午茶、烤肉、住宿组合，有 3 间民宿房。房间展示中出现被子未整理、墙面/空调管相关问题，说明民宿交付标准和日常整理有短板，但勇哥没有把它作为本案核心。
- 开业/任职时间 ASR 粗糙，像是“2025 年 3 月左右”，勇哥复述为“在这上了一年半”。只能保守记录为开了约一年多到一年半。
- 当前流水极低：连线人说“这个月到现在 500 都没有”，勇哥按“营业额归零”处理。
- 房租 3000 元/月，约 100 元/天。员工工资 3500 元/月，约 117 元/天，老板仍按时发。水电气约 100 多元/月，ASR 未给精确数。
- 老板有 3 个，2 男 1 女。投入按五六十万/约 60 万记录。房子原本是五六十年代的老房，之前也有人花钱打造过；现在老板租来继续改造，本质上把一部分装修价值留在了房东资产上。
- 真实客流线索：天气好时爬山的人和路过的人会进小院，但他们常问有没有简餐。现有烤肉、下午茶、住宿的决策成本较高，不一定匹配爬山客即时需求。
- 店内主要就这名员工一个人，复杂菜品或慢出餐产品会压垮执行。门头/招牌现场看不清，外部识别弱。
- 可利用资产包括山景、小院、亭子、狗、员工本人故事、离大学城不远、邻居有两匹马。骑马服务 ASR 中说“可以有”，但需要先和邻居确认安全、收费和责任边界。

## Yongge Question Order

- 先反复核实身份：这是不是你家、你是不是老板、课是老板买还是自己买、你是不是在这里上班。因为员工替老板问诊会影响建议权限和执行边界。
- 问城市，再问业态：卖什么、有没有住宿、民宿房间几间，并让对方展示房间和院子。
- 快速进入经营数字：这个月能卖多少钱，确认本月至今不到 500 后，马上问房租。
- 问固定成本结构：房租 3000/月后，换算成 100/天；再问是不是被老板请的、开多久、工资多少、水电气多少。
- 在发现“流水不到 500 还发 3500 工资”后，再次要求确认不是老板本人，避免基于错误身份做判断。
- 问老板结构和态度：几个老板、男女比例、工资是否按时发、现在是否还能卖到工资钱、季节变化。
- 问总投入和物业来源：投了多少钱，房子是新修还是旧房，房东/前手是否也投入过，周围有几户人，离大学城和景区多远。
- 问真实需求而不是老板想象：平时什么人会上来、他们来了想吃什么、烤肉和茶是否匹配、一个人能否做复杂产品。
- 检查外部可见性：有没有招牌、门头能不能看出来、车能不能开到。
- 最后才给建议，并继续确认执行资源：愿不愿意直播、店名叫什么、地理位置怎么对外说、员工个人故事能否作为内容、邻居马能否变成体验项目、首批顾客能否打折。

## Calculation Logic

- 当前收入口径：本月至今流水不到 500 元。勇哥没有按完整月预测，而是把它作为“几乎没生意”的诊断信号。
- 固定成本下限：房租 3000 元/月 + 工资 3500 元/月 = 6500 元/月，约 217 元/天。加上水电气约 100 多元/月，最低现金支出约 6600 元/月以上，约 220 元/天。
- 亏损下限：如果整月最终仍只有不到 500 元流水，即使按 100% 毛利，也覆盖不了固定成本，固定成本缺口约 6100 元/月以上；真实缺口还要扣食材、损耗、房间清洁、维护等变动成本。
- 保本公式：月保本流水 = 月固定成本 / 综合毛利率。ASR 没有给食材成本和住宿变动成本，所以不能定死毛利率；若仅作教学推算，按 60%-70% 综合毛利，覆盖 6600 元固定成本大约需要 9400-11000 元/月，即日均约 310-370 元流水。
- 投资额五六十万属于沉没成本。由于物业是租来的老房，装修和改造不能简单视为可回收资产；先要验证日常经营能否覆盖工资、房租，再谈回本。

## Diagnosis

- 核心问题不是租金高，而是流量和转化几乎为零。3000 元/月的租金本身不重，但本月至今不到 500 元流水，连一个员工工资都无法覆盖。
- 这是典型的情怀型/梦想型目的地小院：先被环境打动、投入改造，再找生意逻辑。勇哥后段提醒，很多人开店不是因为验证了商机，而是因为一个想法或梦想。
- 位置不是纯深山，但也不是自然街边流量。离大学城和小景区近只是潜在线索，必须把它变成明确到店理由、路线和产品组合。
- 产品和需求错位。爬山客、路过客更可能要快餐、简餐、补给；烤肉、下午茶、住宿更偏计划型消费，不能解决“上山路过想快速吃点”的需求。
- 经营入口弱。门头看不清，线上内容和线下识别都不足；现阶段最有传播力的不是装修本身，而是“一个员工守着山顶小院、狗、山景、老板投资”的反差故事。
- 员工比老板更主动，这是本案的特殊变量。勇哥没有直接劝关店，而是建议用员工执行力做低成本自救测试。

## Recommendation Pattern

- 不再先花钱做大装修、复杂菜品或重资产升级。当前优先级是用最低成本验证有没有人来、来了买什么。
- 立即做内容引流：让员工从当天开始直播，围绕小院、山景、狗、一个人守店、老板投资的小院等真实素材讲清楚“这里是什么、在哪里、能吃什么、能玩什么”。
- 对外定位要具体：重庆璧山、靠近沙坪坝大学城，适合情侣、朋友、大学城周边、爬山后休息打卡。不要只说环境好，要给出可搜索、可导航、可消费的理由。
- 产品上补一个快出餐简餐：如杂酱面、红烧牛肉面或同类标准化简餐。逻辑是匹配爬山客“有没有简餐”的直接需求，同时一个员工也能做。
- 修正基础转化点：门头招牌、路线上山提示、房间卫生和院子展示要能让第一次来的客人放心消费。
- 把周边资源做成体验补充：邻居的马可以沟通成骑马/拍照项目，但必须先确认收费、看护、安全和责任，不应替代主产品。
- 首批流量可以用 5 折/6 折打卡优惠承接，但优惠应绑定可控产品，避免越卖越亏。
- 老板侧要设观察期：如果直播、简餐、招牌和基础体验修正后仍不能覆盖至少 220 元/天左右的固定成本下限，就应考虑止损、转租、改业态或把房子还给房东。

## Reusable Heuristics

- 员工替老板问诊时，先确认身份、权限和老板参与度。没有决策权时，只给低成本、可当天执行的建议。
- 亏损门店的追问顺序可复用：身份 -> 城市/业态 -> 当前流水 -> 房租 -> 人工 -> 水电气 -> 开业时长 -> 投入 -> 位置/客流 -> 顾客真实需求 -> 可执行资源。
- “租金便宜”不能掩盖“流水接近零”。要把房租和人工换算成日固定成本，再看日流水是否能覆盖。
- 目的地门店卖的是到店理由，不只是菜品。山景、宠物、人物故事、路线、体验项目都要服务于“为什么现在要来”。
- 重装修投入只有在能带来未来现金流时才有意义。租来的乡村老房改造，常常会把资产价值留给房东，回本要靠经营而不是装修本身。
- 一人店必须按人效设计 SKU：快出餐、少备货、低损耗、流程短，比菜单丰富更重要。
- 顾客已经开口问的需求，比老板设想的产品更有信号价值。本案中“有没有简餐”比继续强化烤肉更值得优先测试。
- 有故事的非标店适合直播和短视频，但内容必须落到产品、价格、路线和到店转化，否则只是热闹。
- 当前亏损不必一刀切关店；当租金低、资产有特色、员工愿意执行时，可以先做短周期自救测试，再用数据决定继续或止损。
