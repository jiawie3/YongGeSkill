## User Facts Extracted

- 咨询者想接手长春市区一家蜜雪冰城转让店，诉求是让勇哥帮忙算能不能挣钱。
- 门店约30来平方，租金15000元/月，付款方式为年付，押金1个月房租。
- 转让费35万元。转让方称“啥都带了”，设备不动；ASR 中关于是否包含首批食材/现有食材不够清晰，但勇哥按转让费、年租金、押金合计出约54.5万元入场资金。
- 店已经开了约1年半，咨询者称距离第二次装修还有约3年半；勇哥追问该周期是4年还是5年，最终按约5年周期理解。
- 店内当前约4个人，咨询者说每人4000元/月，人工约16000元/月。
- 水电气原句 ASR 不清楚；从勇哥后面“每天固定成本1133元”的计算倒推，除租金和人工外，水电气约3000元/月。
- 咨询者最初说日均约3700元，勇哥计算时用过3800元/日；随后确认上周销售约27000元，实收约27760元，按4周折算约10.8万-11.1万元/月，约3600-3700元/日。勇哥强调这是夏天数据。
- 转让方或系统数据称毛利约55%，咨询者自己也不确定；勇哥先保守用45%测算，后面看点位和外卖占比后认为实际毛利可能只有40%，甚至35%。
- 咨询者最初说外卖和到店一半一半，但没有蹲店验证。勇哥现场看点位后判断该店更可能主要靠外卖，甚至80%-90%外卖。
- 去年数据被转述为实收约140万元，转让方称赚了30多万元、已经回本。勇哥没有直接采信，反问如果真能稳定赚钱，为什么还要35万转出。
- 咨询者没有亲自蹲店，只是看转让方系统截图/报表、问了旁边店和邻居；旁边店流水、竞品流水也都是问来的，没有实地核验。
- 点位在老城区/居民区街边，对面有一个较老的商场，附近有酒店和餐饮，学校较远。勇哥判断这是“人流”不是有效“客流”，位置一般，租金偏贵。
- 咨询者还没有交钱。最终接受建议，表示不干这个店。

## Yongge Question Order

- 先确认咨询目标：问的是哪个店、是否转让、在哪个城市。
- 立刻查基础资产和租赁条件：面积、月租、怎么付、在市区还是县城、押金、转让费。
- 追问转让费包含范围：35万元到底带什么，设备、装修、食材/物料是否包含，避免只听“啥都带了”。
- 查品牌/装修周期风险：店开了多久，距离第二次装修还有多久，周期到底按4年还是5年。
- 先算入场资金：把转让费、年付租金、押金合到一起，确认不是只准备35万元就能接。
- 再问日常成本结构：几个人做、每人工资、水电气、毛利率、外卖和到店比例。
- 再问流水口径：一个月/一天卖多少钱，并把上周实收折算成月流水和日流水。
- 对数据来源追问到底：去年报表还是今年报表，系统截图还是实收，是否逐笔核对，是否蹲店看过。
- 追问运营验证动作：有没有看一天多少杯、外卖多少单、进店多少单，而不是只问别人。
- 追问转让来源和转让原因：哪里看到的转让信息，转让方为什么转，转让方说是否已经回本。
- 要求视频360度看点位：看商场入口、周边居民区、老城区街道、学校距离、附近餐饮和竞品。
- 最后把点位、季节、外卖占比和租金放回模型，推翻前面基于卖方数据的乐观测算，给出不接手的建议。

## Calculation Logic

- 入场资金：转让费350000元 + 年付租金15000*12=180000元 + 押金15000元 = 545000元。勇哥口径为“看下来五十四万五左右”。
- 固定成本/月：租金15000元 + 人工4*4000=16000元 + 水电气约3000元（ASR 未清楚说出，按勇哥日固定成本倒推）= 约34000元/月。
- 固定成本/日：34000/30=约1133元/日。
- 表面流水：咨询者说日均3700-3800元；按上周27000元折算，27000*4=108000元/月，约3600元/日。勇哥强调该数据发生在夏天。
- 乐观毛利测算：按3800元/日、45%毛利，毛利额为1710元/日；扣掉固定成本1133元/日，约剩577元/日，约17310元/月。
- 表面回本期：只用35万元转让费除以约1.7万元/月，约20个月回本。这是勇哥按转让方数据做出的第一层测算，不代表认可接店。
- 如果按完整入场资金54.5万元测算，即使沿用乐观月利润约1.7万元，也要约31个月以上回本；这个口径比只算转让费更能反映现金压力。
- 风险修正：若实际外卖占比很高，毛利从45%降到40%甚至35%，夏季3600元/日对应毛利额约1440元/日或1260元/日，扣固定成本1133元/日后只剩约307元/日或127元/日。再遇到长春冬季冷饮销量腰斩，门店会从微利变成亏损。

## Diagnosis

- 核心问题不是转让费单点高低，而是咨询者拿到的数据不可采信：卖方报表、转让方口径、邻居和旁边商户说法都没有通过蹲店、逐笔核对、外卖单量、进店单量和杯数验证。
- 该店用夏季数据看似有利润，但品类是冷饮茶饮，城市在长春，冬季长、温度低。勇哥认为夏季流水至少要考虑腰斩，不能拿旺季周流水直接年化。
- 点位质量偏弱：街边老城区、居民区和旧商场带来的是路过人流，不一定是购买客流；学校距离远，商场客流一般。租金15000元/月对这个点位偏贵。
- 咨询者声称外卖和到店一半一半，但现场点位不支持强到店判断。勇哥判断更可能高度依赖外卖，而高外卖占比会稀释毛利，系统显示的55%毛利不能直接当可用利润。
- 转让方叙事有矛盾：如果去年实收140万、赚30多万、已经回本，又愿意35万转让，需要反推其真实动机，而不是接受“合伙人去外地/没空”的表面理由。
- 咨询者没有耐心蹲店，却想花几十万元接店。勇哥把这视为经营能力风险：接店前都不愿意做数据调研，接手后更难管住经营细节。

## Recommendation Pattern

- 本案建议是不接手。咨询者未付款时应直接止步，不要被“年销140万、35万转让、老板已回本”的话术带走。
- 若类似店还想继续评估，必须先做实地验证：看完整营业日的进店单、外卖单、杯数、分时段客流、系统实收和外卖后台，并把数据逐笔对上。
- 对茶饮/冷饮转店，不能只看夏季周流水。要把旺季、淡季分开，尤其北方城市要单独测算冬季销量下降后的净利润。
- 对高外卖占比门店，要用扣平台成本后的实际毛利重算，不要直接采用品牌系统或转让方口头给出的55%毛利。
- 对年付租金店，现金预算要把转让费、年租、押金一起算。只讨论“35万转让费多久回本”会低估真实回本周期和现金占用。
- 对转让方给出的赚钱故事，要反问“既然赚钱为什么转”，并用真实数据验证原因；不能用转让方故事替代尽调。

## Reusable Heuristics

- 人流不等于客流。居民区、老商场、酒店和餐饮路过人多，不代表能形成茶饮到店购买。
- 转店先验数据，再验点位，最后才谈价格。没有蹲店、没有核账、没有外卖/进店/杯数拆分，就不要接。
- 旺季数据不能直接年化。冷饮茶饮在长春这类北方城市，要先把冬季销量腰斩甚至更低的情形放进模型。
- 高外卖占比会同时拉低毛利和经营确定性。到店毛利、系统毛利、外卖扣点后的实际毛利要分开看。
- 表面20个月回本已经不短；如果只算转让费都要20个月，按总入场资金和淡季修正后通常风险更大。
- 转让方说“去年赚了三十多万、已经回本”不是利好本身，反而要追问真实转让动机。
- 接店前不愿蹲店的人，不适合接需要强运营和细账核对的小店。
