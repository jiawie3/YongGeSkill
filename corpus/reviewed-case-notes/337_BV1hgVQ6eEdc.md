## User Facts Extracted

- 案例阶段：计划开业/建设中，不是已经经营亏损的门店。用户没做过餐饮，原先做服装，想让勇哥帮忙看能不能继续投。
- 城市与位置：东莞大朗，一个镇级大公园里面。用户描述可用区域约三千多平，另有上方一千多平烧烤场地；整个公园面积很大，ASR记为“几十公顷”。
- 项目构想：在公园内做复合业态，包括便利店/小超市、奶茶、钓虾池、烧烤/营地、游戏机、娃娃机、台球、儿童娱乐设施、铺草坪和造景等。便利店和部分娱乐设施被用户定位为引流或配套，不指望单独盈利。
- 场地关系：这片区域不是用户直接租来的，用户说朋友租了10年；用户与朋友签了合同，朋友不收固定房租和押金，按营业额抽10%。也就是月流水每10万元，先给朋友1万元。
- 投资责任：用户主要或全部出钱建设，朋友不出资，可能帮忙经营。若生意不好，用户口头表示走了就把固定装修/造景留下，等于重资产沉在朋友或场地方。
- 预算口径：用户先说下面区域大约50万元，不含烧烤后可能更多；又提到70万元，勇哥追问后按“60万元肯定能搞定、包含设备”作为核心投资额。已投入金额ASR处出现“二三十万了吧”，但后文用户又说还没有多少钱放进去，需视为粗糙未确认。
- 奶茶部分：计划自营。用户找到一个不做的奶茶店，对方把设备和技术一起转给他，报价约3万到4万元，已付几千元定金，安排人学了半个月。勇哥按二手制冰机、开水机、果糖机、冷藏柜等估算，认为二手设备加起来不到1万元，3万到4万元偏贵。
- 公园现状：用户承认现在流量不多，周末也只是三三两两，不是扎堆人群。公园过去可能火过，但因转包/换公司及游乐场安全事故停了约半年，当前重新开放和规划，其他项目前面还在拆、重新招标或改造。
- 入口与交通：用户区域离公园入口还要往前走并转一下，约100多米。工作人员车辆能进，普通顾客车不能开到里面，也没有可出租/接驳的电动车等配套。勇哥判断这是烧烤、烤肉、亲子消费的明显痛点。
- 用户收益预期：用户理想状态是一年赚过10万元就开心，不以高利润为目标。其决策动机夹杂对朋友的信任、重感情和“试一下”的心态。

## Yongge Question Order

- 先让用户说问题和镜头看场地：确认用户没做过餐饮，要求展示项目空间和自己的想法，而不是先听宏大规划。
- 追问空间与业态：这三千多平是否全部由用户打造，里面打算做什么，逐项确认奶茶、钓虾、烧烤、免费娱乐、儿童设施、便利店等。
- 追问城市与场地成本：问在哪个城市、整个场地租下来多少钱；继续确认是否没有固定房租、是否按营业额10%分成、有没有押金。
- 追问场地性质与权属：确认这是公共公园，不是朋友自己的园子；再追问这片区域是朋友的还是朋友租的、租了几年、是否签合同。
- 追问投入预算和已投金额：问装修准备投多少钱、是否包含设备、现在一共投进去多少钱；用60万元作为核心风险锚点。
- 追问用户能力与品类能力：问奶茶是否加盟还是自营、技术会不会、设备加技术多少钱、付了多少定金；再问用户以前干什么，判断餐饮经验不足。
- 要求现场走动验证位置：让用户走到公园入口和外侧，观察从外面到项目的动线、距离、可见性、人流密度和周末客流，而不是只听用户说“公园位置好”。
- 追问公园历史与配套：问以前为什么关、关了多久、是否出过安全事故、整个公园多大、是否允许搭帐篷、草坪人多不多、车是否能进来。
- 追问合作逻辑：问朋友是否会帮经营、为什么朋友自己不投钱、不自己干；把“信任朋友”转回“信任市场和客流”。
- 追问收益目标：听到用户说一年赚10万元，立刻反算投资回收期，逼用户面对60万元重资产对应的回本逻辑。
- 最后给动作指令：不是继续讨论怎么造景，而是要求所有装修和投入先停，观察公园真实发展和人流，再决定是否用小投入重启。

## Calculation Logic

- 场地“无房租”不等于低成本：固定租金为0、押金为0，但朋友抽营业额10%，本质是变动租金/渠道抽成。月流水10万元时抽成1万元；月流水30万元时抽成3万元。它会直接压低低价烧烤、便利店和奶茶的贡献毛利。
- 核心投入账：用户计划总投资约60万元，且很多钱会变成草坪、造景、池子、固定装修等不可搬走资产。若失败后“留下”，这部分不再是可回收设备，而是给场地增值。
- 回本账：用户理想年利润仅10万元，折合月利润约0.83万元。60万元总投入 / 10万元年利润 = 6年简单回本；若总投入按用户口径上探到70万元，则同样利润下约7年回本。这个算法还没扣除设备折旧、淡旺季、人工、水电、维护、食品损耗、用户自己的时间和合同风险。
- 已投账：ASR有“二三十万了吧”的说法，但后文又说“还没有多少钱放进去”，不能当成精确事实。勇哥的实际处理逻辑是：不管已投多少，已发生的沉没成本先认掉，从今天起停止继续加码。
- 奶茶转让账：对方要3万到4万元交设备加技术，用户已付几千定金。勇哥用二手设备清单拆解，认为制冰机、开水机、果糖机、冷藏柜等二手价值远低于报价，先不要继续为“技术”和设备溢价买单。
- 保本流水无法从ASR完整计算：没有可靠人工人数、工资、水电、原料成本、毛利率、客单价、日单量和营业天数。此案不能编保本流水，只能确认投资回收期过长、客流未验证、分成抽成和重资产沉没风险过高。

## Diagnosis

- 主要问题不是菜单，也不是装修方案，而是“没有被验证的消费人群 + 重资产建设 + 朋友不共担风险”。
- 用户把“公园面积大、没有竞争、朋友靠谱”当成优势。勇哥的判断相反：没有人、车进不来、动线远、消费场景不成立时，没有竞争往往代表没有需求。
- 公园人流不能直接等于餐饮客流。早上6点到9点多是锻炼、跑步人群，最多买瓶水；上午和工作日中午人少；周末亲子家庭更多是自带帐篷、椅子、食物去露营和玩公园，不一定会到里面买烧烤、奶茶或游乐项目。
- “便宜”和“免费玩”不是充分引流理由。用户想用低价烧烤、普通奶茶、免费娱乐把人吸进来，但勇哥指出现在已经不是便宜就一定有人来的阶段，产品、环境、营销和地理位置要同时成立。
- 奶茶需求弱。年轻人来公园不一定专门喝奶茶，且奶茶点位在里面，品牌也不是强品牌，最多被水和即时补给替代。
- 公园游乐和多业态堆叠风险高。勇哥认为公园里的游乐场多数难长期做，天气、停留目的和客群消费意愿都限制强；本案又叠加钓虾、烧烤、奶茶、便利店、儿童娱乐，容易变成杂乱投入而不是一个明确模型。
- 合作结构不对等。朋友控制资源、抽流水10%、不投钱，用户承担60万元建设和经营风险。若项目不成，用户走人留下固定资产，朋友和场地方受益，用户承担亏损。
- 公园自身还没有成熟。它刚恢复、重新招标和规划，曾因安全事故停摆约半年，当前客流没有恢复。此时抢先重投，不是抢机会，而是在替不确定的公园发展买单。

## Recommendation Pattern

- 立即暂停所有装修、造景、设备尾款和新增订单。对外可以说“方案重新整理、先观察”，但动作上先停止继续烧钱。
- 把已花的钱当沉没成本处理，不要因为已经投了定金、挖了池子或签了合同，就继续把60万元投满。
- 先观察1个月、2个月、3个月到半年，看公园是否真正恢复稳定客流：周末是否扎堆、平日是否有人、停车和接驳是否解决、其他项目是否开起来、是否形成类似网红公园的自来流量。
- 如果半年后公园还是没有人，不做，能清掉就清掉，直接止损。
- 如果公园后续真的发展起来，再重新评估，也不要做大杂烩。只选择一个更贴近公园场景的轻模型，例如营地/烧烤方向，先用可移动设备和小投入验证，勇哥给出的量级是几万元到十万元，而不是60万元造景重装。
- 重新谈合作前，要把投入上限、资产归属、退出机制、朋友是否出资或承担亏损、10%流水抽成、停车/接驳/食品经营和安全责任写清楚。朋友若仍一分钱不投，只抽流水，就不能把“兄弟感情”当投资依据。
- 奶茶设备转让要暂停尾款，重新按二手设备真实价值核价；不要为了一个普通奶茶技术包付3万到4万元。
- 营销不能作为坏位置的补药。抖音、团购和网红拍照只能放大已有产品和客流，不能保证把人从外面持续拉到车进不来、入口远、配套不成熟的公园深处。

## Reusable Heuristics

- “零房租”先拆成两层：有没有固定租金，以及是否存在流水抽成、装修沉没、不可搬走资产。没有固定租金但要投入重装，风险可能比租金更大。
- 看公园、景区、营地项目，要先分清人流和消费流。锻炼、散步、露营、自带食物的人，不自动等于奶茶、烧烤和游乐顾客。
- 选址不能只看面积和独占权。一个空场地“全部你说了算”，如果入口远、车进不来、没有人，就没有经营价值。
- 朋友合作要看风险共担。资源方不出钱、只抽流水、失败后还留下固定资产时，投资人承担的是单边风险。
- 重资产项目前置验证比装修更重要。先数人、看动线、测停车、测核销和消费转化，再投钱；不能靠“试一下”直接上几十万。
- 高风险餐饮项目不能接受过长回本期。计划投60万元、理想年赚10万元，简单回本6年，已经暴露投资逻辑薄弱，更不用说餐饮折旧和不确定性。
- “没有竞争”常常是需求不足信号。真正要问的是顾客为什么来、来的人为什么买、买完是否复购，而不是附近有没有同类。
- 多业态堆叠不是能力。便利店、奶茶、钓虾、烧烤、游乐、造景同时上，会放大管理复杂度和资金消耗；没验证前应缩成一个最有消费理由的模型。
- 团购或短视频数据要看核销和到店转化。卖出券或视频热闹不等于真实消费，必须看实际有多少人走到店里、核销、复购并覆盖成本。
