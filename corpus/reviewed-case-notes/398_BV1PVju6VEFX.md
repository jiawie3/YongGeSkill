## User Facts Extracted

- 咨询者不是老板，原为厨师长，视频当年6月左右被提为店长，31岁；他想把店救起来以保住工作。老板平时很少到店，这家店属于老板副业。
- 门店在杭州，ASR 店名听作“关内牧羊人”，主打羊肉小火锅/羊肉汤锅，一人一小锅；有牛肉、海鲜和少量凉菜/小吃，但没有炒菜。
- 门店位于一个园区/旧改区域，位置偏，附近只有一个地铁口；自然流量弱。工作日因周边写字楼/上班人群反而好一些，周末差。
- 面积约1000平，约20个包间，两层结构，没有大厅，没有普通卡座。一楼主要是入口、前台和楼梯，经营座位主要是包间。
- 包间结构不适合散客：最小三人包间只有一个，四人以下顾客经常只能坐更大的包间，空间利用率低。
- 当前平均日流水约1万元，对应月流水约30万元。咨询者说早期月流水 ASR 听作约10万，现在做到30多万，但仍亏。
- 报告毛利率约60%，勇哥质疑“有这么高吗”，但现场先按60%参与计算。
- 客单：普通人均约80-90元；商务/高消费场景可到约500元/人，但这类订单一周大约一两次，占比不高。
- 租金约15万元/月，约5000元/日。
- 人员前后场合计22人。人工口径 ASR 粗糙：字幕一处像13万元/月、平均工资6000元，但勇哥现场计算用了约7333元/日人工，相当于约22万元/月；另有运营/销售主管，销售主管月薪约8000元，是否已包含在22人内未确认。
- 水电夏天约8000-10000元/月，勇哥按约333元/日计。
- 已知经营亏损：勇哥现场用毛利6000元/日，扣租金5000元/日、人工7333元/日、水电333元/日，得出每天亏约6666元，约19.98万元/月；还未完全计入折扣费等，实际可能更高。开场“亏30万/月”更像概括口径。
- 线上运营：有全平台运营，每天发视频；每天中午12点到2点由销售主管直播一场。直播间人数有时十来人，有时几十人，销售转化不稳定，甚至播不出钱。
- 老板原定位想做商务接待；咨询者认为周边自然流量少、老板需求少，所以商务接待做不起来。勇哥进一步指出商务接待不是靠自然流量，而是产品场景、客群和区域需求都没对上。
- 老板准备在一楼扩建大厅，约十来天后动工，成本未确认。咨询者说老板还想加烧烤，原因之一是园区背面开了烧烤店，且想做夏天生意。

## Yongge Question Order

1. 先确认来电者是不是老板；不是老板时继续追职位、权限、是否经过老板同意、何时从厨师长转为店长。
2. 问城市、店名、品类和吃法：是羊肉串、羊肉汤还是火锅；确认是一人一锅的小火锅。
3. 立即追经营基本数：现在一天卖多少钱、毛利率多少、面积多大、是不是全包间、有无大厅。
4. 按日损益追成本：租金多少、几个人、人工多少钱、水电气多少钱；把月口径即时除以30天换成日口径。
5. 现场先算亏损：1万流水、60%毛利，毛利约6000元/日；再扣租金、人工、水电，推出每天亏约6666元、月亏约20万，并追问是否还有折扣费。
6. 问老板管理状态：老板多久来一次、是不是副业、老板做什么、店长是否全权负责、为什么换掉前任管理者。
7. 让咨询者360度看周边商圈：园区、地铁口、是否郊区、几楼、几层、有无自然流量。
8. 追线上运营是否真正带客：谁做运营、是否直播、谁在播、每天播几场、播多久、直播间多少人、视频发什么内容。
9. 巡店看空间和产品：一楼、二楼、包间数量、菜单、是否有炒菜、菜品是羊肉/牛肉/海鲜还是中餐宴请菜。
10. 问客单和目标客群：普通人均、商务高客单能到多少、高消费订单频次、到底是要做高端接待还是平民消费。
11. 继续追包间适配性：工作日/周末哪天好、一天用多少包间、小包间几个人、三人/四人包间数量，验证散客是否被大包间浪费。
12. 最后给结构性诊断：定位不清、全包间无大厅、产品不适合接待、人效过低、营销救不了模型；再给改菜品、改卡座、改人效和向老板汇报的顺序。

## Calculation Logic

- 当前流水：约1万元/日，按30天约30万元/月。
- 报告毛利率：约60%。按此口径，日毛利约1万 * 60% = 6000元。
- 租金：15万元/月，约5000元/日。
- 人工：勇哥现场算账采用约7333元/日，折合约22万元/月；但 ASR 对原始人工月额听写有冲突，实际工资需复核。
- 水电：夏天约8000-10000元/月，勇哥按约333元/日口径。
- 勇哥现场亏损式：6000元日毛利 - 5000元租金 - 7333元人工 - 333元水电 = -6666元/日，约-199800元/月。
- 这还未完整纳入折扣费、营销成本、损耗等，故“月亏二三十万”是合理风险区间，但精确数只能确认到勇哥现场的约20万/月经营亏损口径。
- 保本流水：已知日固定/半固定支出约5000 + 7333 + 333 = 12666元；按60%毛利率倒推，12666 / 0.6 ≈ 21110元/日。勇哥口径为保本线约2.1万元/日。
- 当前日流水1万元离保本约2.1万元差约1.1万元/日；在模型不改的情况下，单靠直播和套餐很难补上。
- 人效：1万元日流水 / 22人 ≈ 454元/人/日。勇哥据此判断“人效非常低、人力成本非常高”。

## Diagnosis

- 核心不是单个菜品或直播话术问题，而是店型模型错配：1000平、20个全包间、无大厅/卡座，却实际靠工作日周边上班族贡献生意，空间没有服务主要客群。
- 定位不清：说普通消费，店里全是包间、没有大厅，普通顾客进入门槛高；说高端接待，又主要是羊肉一人一锅，没有炒菜和正式中餐宴请结构，高端也高端不起来。
- 商务接待场景不成立：接待客人通常要聊天、谈事、体面点菜，而一人一锅需要客人自己烫菜，服务员在旁边也干扰私密谈话；这与正式宴请/商务谈事的需求冲突。
- 产品带宽尴尬：普通羊肉人均80-90元，海鲜/高档菜又能把人均拉到约500元。请客者只点羊肉显得不体面，点海鲜又太贵，导致高消费客群难以稳定选择。
- 包间供给不匹配实际订单：二三四人散客多，但小包间少，顾客被迫占用大包间；20个包间做不满，坪效被硬装和大桌拖低。
- 位置偏、自然流量弱，但工作日有写字楼上班族需求。店没有用卡座、双人/三四人套餐去承接这类高频需求，反而把空间锁死在低频商务接待上。
- 人效过低：22人支撑1万元日流水，约454元/人/日，人工结构已经吞掉贡献毛利。
- 运营岗位和直播不是第一矛盾。直播间人数少、转化不稳定，本质上是产品、空间和定位没有形成清晰购买理由；在模型错的时候加流量，只会加速亏损或买差评。
- 老板认知和决策链是风险：前任店长提过大改但被否决；老板仍想一楼扩建并加烧烤，像是看到别人做什么就跟什么，没有先回到自身客群、场景和保本模型。

## Recommendation Pattern

- 先定方向：如果真做高端海鲜/商务接待，就要彻底围绕高端海鲜和正式宴请设计，不要继续用羊肉小火锅当主体；如果不做高端，就要回到大众可消费的中餐/接待型餐饮。
- 勇哥更明确推荐第二条：必须加中餐、加炒菜，把羊肉作为特色，而不是全部销售理由。厨房没有炒锅不是结论，需要想办法改造或用其他设备解决。
- 改空间模型：拿使用率低的大包间撤掉大桌，改成卡座/小桌；条件允许就打通两个包间，不允许打墙也可以先在包间内改卡座。目标是补双人、三人、四人消费场景。
- 围绕工作日上班族重做产品和场景：上双人餐、三到四人餐、四人餐，把直播间和短视频的展示场景从“豪华包间接待”改成“普通人也能来吃”的场景。
- 优化人工和岗位：在日流水只有1万元时，22人和多岗位配置过重。要按新营业面和新桌型重排班、减岗位、提高人效。
- 一楼如果要扩建，大方向应是补大厅/卡座和中餐能力，而不是加烧烤。烧烤只是看到附近有人开就跟，不能解决现有定位和保本问题。
- 对老板汇报时不能只说“要改位置、加炒菜、加卡座”，要先讲为什么：当前工作日客群来自上班族、全包间挡住散客、接待场景不适合一人一锅、保本线需要2.1万/日而现在只有1万/日。
- 如果老板拒绝结构性调整，只愿继续扩建或加烧烤，店长没有运营层面的解法。该尽到汇报职责，之后要接受这家店继续亏损甚至倒闭的结果。

## Reusable Heuristics

- 非老板来咨询，先问身份、权限和决策链。店长能不能改模型、动装修、减人、加品类，决定建议是否能落地。
- 大店先把月口径改成日口径：日流水、日毛利、日租金、日人工、日水电，再算每天亏多少和保本日流水。
- “全包间”不是高级的同义词。若主要客群是二三四人散客或工作日上班族，没有大厅/卡座会直接损失高频需求。
- 商务接待不只看装修和包间，还要看产品是否适合谈事、是否有正式点菜结构、是否体面、服务是否不打扰。
- 定位不能同时摇摆在“人均80-90的羊肉小火锅”和“人均500的高端接待”之间。客群、菜单、空间和服务必须指向同一个购买理由。
- 当工作日好于周末时，要反推真实客群可能是周边办公人群，而不是想象中的高端宴请客。空间和套餐应优先承接真实客群。
- 人效可快速体检：日流水 / 在岗人数。如果每人每天只创造几百元流水，而毛利率又只有常规水平，人工大概率压垮模型。
- 营销前先修模型。产品、空间、价格带和服务场景不成立时，直播和投流只是在把错误模型推给更多人。
- 看到别人做烧烤、海鲜、直播就跟，是老板认知风险。新增项目必须回到本店客群、厨房能力、空间利用率和保本流水验证。
- 给老板提改造建议要讲逻辑链：现状数据 -> 保本差距 -> 客群证据 -> 空间/菜品错配 -> 具体改造动作。只提结论通常会被否决。
