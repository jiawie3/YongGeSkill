## User Facts Extracted

- 阶段：计划开店，尚未签定租赁/管理方方案；不是已营业亏损店。
- 城市和场景：广州白云区人和镇，一个较偏的教育园区，园区约6万多平方米，有课室、宿舍和饭堂。培训类型包括艺考、高考/成考、公务员和职业类培训。
- 点位：拟租其中一栋楼的露台/天台，外部约600平方米；另有室内3个房间，每间约60平方米，室内合计接近200平方米，总可使用面积约800平方米。动线需要学生或客人坐电梯上楼。
- 项目设想：室内做私厨接待/包间，天台做咖啡奶茶；对话中又加入集装箱书吧、简单健身房、类似会所的配套。核心品类没有收敛。
- 租金方案：用户自己提出约5000元/月保底租金加营业额5%分成，管理方还未同意；无入场费。押付方式 ASR 粗糙，能确认的是押金约5000元口径被提到。
- 用户身份：用户是物业/招商方之一，称园区99%机构是自己招回来的，对场地熟悉，但对餐饮工程条件和成本没有核清。
- 经营主体：自营，3个合伙人，年龄约30岁上下。三人中有人以前做过餐饮但失败过；技术计划靠请厨师，前期还想找兼职厨师。
- 人员计划：一个股东可能参与经营；水吧计划约2个全职，再加若干兼职。兼职数量未明确，用户还提到可找园区学生兼职，但勇哥判断这类学生并不稳定。
- 预算：用户预计装修总投入10万元以内，包含3个房间内部装修。勇哥认为仅白墙地砖不算可经营装修，800平方米、包间、厨房、户外造景、设备、烟道、燃气/电力、集装箱吊装等都会远超10万元。
- 工程条件：有天然气的说法，但烟道不存在或未建；用户想在天台直接排烟，勇哥判断不可行，烟道要上楼顶。场地在5楼口径出现过，燃气上楼、电力增容、厨房排烟、设备上楼/吊装都需单独核价。用户只知道类似“370V/380V”的电压说法，不知道可用电量/千瓦数。
- 客群和现有供给：园区当前约1000人，用户称下半年高峰可能到2000-3000人，但部分客户还在谈。园区已有4个食堂/餐饮点，其中2个较大、2个偏小吃/小餐馆；便利店日销约3000-4000元。食堂日销“2000-3000元”被提到，但 ASR 未确认是单店还是整体口径。
- 用户收入预期：私厨接待预计每月约3万-5万元；水吧/茶饮预计日流水约1000元。宣传计划主要依赖园区机构老板帮忙通知学生，再用书吧、健身房等配套吸引人。

## Yongge Question Order

- 先打断发散介绍，要求“问什么答什么”，第一步确认城市和具体场景：广州白云区、教育园区、培训机构类型。
- 追问租赁对象和面积：租的是露台还是室内，600平方米露台、3个房间、总面积约800平方米。
- 立刻问租金：多少钱、保底加分成是谁提出的、为什么不直接固定租金、管理方是否同意、押付方式和是否有入场费。
- 识别用户角色：既然用户是招商/物业方，就反问他是否应该更懂场地条件、食堂需求和工程约束。
- 明确模型：做什么品类，是加盟还是自营；私厨接待、咖啡奶茶两块业务分别是什么。
- 查团队能力：几个合伙人、有没有餐饮经验、厨师从哪里来、兼职厨师是否现实、是否有股东实际在店里经营。
- 查装修和基础工程：装修预算多少、有没有天然气、有没有烟道、能不能直接排烟、现有装修是否只是白墙地砖。
- 查物流和施工可达性：电梯有多宽，设备是否能上楼，集装箱是否要吊装，为什么不用更低成本棚架，是否涉及违建和成本。
- 查餐饮硬条件：厨房、烟道、燃气、电力增容、可用电量多少千瓦、烟道做到楼顶多少钱、是否允许办食品经营相关证照。
- 粗算固定成本：按人工、水电气、毛利率估出日保本流水，再问用户是否算过账、认为这里能卖多少钱。
- 查获客和真实需求：怎么宣传，是否只靠机构老板转告学生；书吧、健身房、私厨接待分别服务谁。
- 现场看商圈和动线：周边是否偏、是否都是培训机构、学生人数和季节性、为什么不在一楼开、已有食堂和便利店卖多少。
- 最后回到核心判断：这个需求是不是学生真实需要，天台动线是否会让人上来，接待需求到底多大，预算和能力是否匹配。

## Calculation Logic

- 勇哥现场给的经营假设：人工约15000元/月，水电气约6000-7000元/月，毛利率按55%算；保底租金约5000元/月。
- 不含营业额5%分成时，月固定成本约26000-27000元。按55%毛利倒推，月保本流水约4.7万-4.9万元，折合日保本约1580-1636元。ASR 中出现的“1636”基本对应这个日保本口径。
- 若把5%营业分成也视为随销售额扣除，实际贡献率约从55%降到50%，月保本流水会升到约5.2万-5.4万元，日保本约1730-1800元。这个是基于 transcript 数字的复算，勇哥现场主要强调日保本约1636元。
- 上述保本还没有完整纳入折旧、证照、推广、损耗、维修、集装箱吊装、烟道、燃气上楼、电力增容、户外维护和股东工资，真实安全线应更高。
- 投资预算口径不稳定：用户预算装修10万元以内；勇哥先指出装修、设备、厨房、烟道、包间和户外造景会显著超支，ASR 中出现“装修30万、设备5万”和“至少27万、三人一人9万”的不同粗估。可确认的逻辑是：10万元无法覆盖800平方米复合业态开业条件。
- 收入预期和保本线的矛盾：用户预期私厨月3万-5万元，水吧日流水约1000元，理论上如果全部实现可覆盖基础保本；但这些收入来自未验证需求，且要在已有食堂、便利店和外卖竞争下让学生专门上天台消费。
- 需求校验参照物：园区当前约1000人，已有地面食堂/餐饮点和便利店。若更便利的食堂日销只有2000-3000元口径，天台项目要稳定做到日保本1600元以上，需要非常强的差异化和动线理由，而 transcript 中没有证据支持。

## Diagnosis

- 主因不是单一租金高，而是“需求假设未验证 + 点位动线错误 + 工程成本漏算 + 业态发散”。用户是看见便宜天台和大面积后倒推项目，而不是从学生高频需求倒推品类和位置。
- 天台餐饮天然吃亏：培训学生的高频需求是方便、便宜、快，已有食堂、便利店和外卖可替代；让学生专门坐电梯到楼顶买奶茶、吃私厨或健身，看不到强动机。
- 客群数量被高估：当前约1000人不是1000个稳定顾客；培训机构人群有周期，艺考/培训批次更换后需要重新触达，不能把园区人数直接当成可消费流量。
- 私厨接待需求不实：用户说有物业/老板接待，但没有明确接待频次、客单和稳定订单。兼职厨师加低预算包间，很难支撑“接待”场景的出品和环境要求。
- 多业态叠加反而暴露没有核心：咖啡奶茶、私厨、书吧、健身房、会所分别需要不同空间、设备、运营能力和客群理由，组合在一起没有形成明确赚钱模型。
- 工程和证照是硬风险：烟道、燃气、电容量、增容费用、设备上楼、集装箱吊装、食品经营许可、消防/双证是否能办，都还没核实。任何一项不成立，项目就不能按设想落地。
- 团队能力不足：有餐饮失败经验但没有成熟餐饮团队；兼职厨师、学生兼职、一个股东兼顾经营，都不足以支撑800平方米复合餐饮项目。

## Recommendation Pattern

- 结论：按当前方案不要做，租赁和装修都应暂停。这个项目不是“再优化一下”能成立，而是起点就把需求、位置、预算和施工条件想反了。
- 如果一定要在园区做餐饮，应先回到一楼或学生必经动线，用小面积、单品类、高频刚需验证，例如早餐、特色快餐、麻辣烫/粉面等；不需要800平方米，50-200平方米已足够测试。
- 先用已有食堂和便利店数据校验天花板：按学生人数、食堂日销、便利店日销、外卖可得性，判断园区还能多容纳多少餐饮流水，而不是从“高峰可能两三千人”推想消费。
- 任何上楼餐饮项目，签约前必须拿到书面工程答案：烟道是否允许、做到哪里、报价多少；燃气能否上楼；电容量多少千瓦、是否要增容；设备能否进场；消防和食品证照能否办。
- 需求验证必须先于装修：私厨接待要先拿到明确订单频次和预算，水吧要先证明学生愿意上楼消费，不能把机构老板“帮忙宣传”当成获客系统。
- 控制业态：不要同时做餐吧、书吧、健身房、会所和私厨。先证明一个高频品类能赚钱，再谈配套。

## Reusable Heuristics

- 便宜租金不能抵消差动线。餐饮尤其是学生餐饮，便利性往往比环境想象更重要。
- 园区/学校/培训场景不能只看“总人数”，要看现有食堂流水、人群停留周期、外卖替代、是否住校、是否愿意绕路。
- 上楼餐饮先问工程四件事：烟道、燃气、电容量、证照。任何一项没答案，都不能先按装修预算算项目。
- 大面积复合业态常见风险是“每个需求都像有一点，但没有一个需求足够强”。问诊时要不断追问：到底服务谁、为什么现在没有被满足、为什么一定来你这里。
- 私厨/接待类项目不能用“偶尔有接待”立项，必须有稳定接待频次、客单、决策人和出品能力。
- 合伙人餐饮经验不足时，不要从800平方米、重装修、重工程项目开始；先用小面积、低固定成本、单品类验证。
- 保本流水要用贡献毛利算，不是用营业额感觉算；有营业额分成时，它会降低真实贡献率。
- 现有更便利业态的流水是上限参照。如果一楼食堂和便利店流水都不高，楼顶项目必须拿出更强证据，否则默认需求不足。
